## 用途

这个 Skill 用于记录营销互动首页可复用 Zero 组件规范和素材链接，帮助低保真布局、高保真重建和组件复用阶段查询顶部通栏、常驻功能信息区、京豆、红包、资源位 Icon、进度激励区和 Feeds 的链接、尺寸、状态与使用规则。

## 适用场景

- 首页布局、低保真绘制或高保真重建阶段需要复用已有 Zero 标准组件。
- 需要确认组件尺寸、状态、可改项、圆角、裁切边界和布局规则。
- 需要避免重复绘制、错误抠图、比例变形或组件被父容器裁切。

## 输入

- 当前页面需要使用的组件类型。
- 组件所在页面或模块的尺寸、状态、变体和可改项需求。
- 来自 `games-homepage-layout`、`games-lofi` 或 `games-hifi` 的组件查询请求。

## 输出

- 可复用 Zero 组件链接。
- 组件用途、尺寸、状态和可改项。
- 组件复用、缩放、裁切、圆角和布局边界规则。
- 对应阶段可使用的组件引用说明。

## 依赖资料

- `SKILL.md`：组件清单、Zero 链接和复用规则。
- `references/rules.md`：组件复用、缩放、裁切、圆角和布局边界规则。
- Zero 设计稿或组件库链接。
- `games-lofi`：圆角等级与低保真组件表达规则。

## 边界和风险

- 本 Skill 只提供组件规范和复用边界，不决定组件是否应该出现在页面中。
- 不允许只修改外框导致内部裁切。
- 不允许出现组件半截、裁切、意外溢出或比例变形。
- 已有标准组件应优先复用，不应重复绘制或从高保真 JPG 中抠取。
- 组件链接、节点 ID 和标准组件状态如有变化，需要维护人同步更新。
