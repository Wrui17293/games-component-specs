# 组件规范清单

执行本规范时，先读取 `references/rules.md`，再查询或复用具体 Zero 组件。

## 总规范
- 除特殊标注外，组件均允许等比缩放，默认 `fitMode=contain`。
- 不允许只修改外框导致内部裁切，不允许出现半截、裁切或意外溢出。
- 可用渲染节点ID、禁止使用的标注节点ID、推荐宽高与安全区
- 组件圆角必须引用 `games-lofi` 的 `R1–R6` 等级，不在本文件重新定义圆角数值。
- 已有标准 Zero 组件保持原有圆角；新建或可调整组件必须标明所用圆角等级。

## 常驻功能信息区
- 用途：承载账户、等级、体力、次数、资产、聊天、商城、兑换等常驻信息或功能入口。
- 布局关系：固定使用“叠加”，组件自身不参与页面纵向排布。
- 组件类型：状态信息项 / 侧边功能入口 / 功能入口组。
- 基础组成：Icon、主文案；数值、副文案、角标和跳转箭头按 PRD 选用。
- 组件状态：默认 / 按下 / 高亮 / 禁用。
- 可改项：Icon、文案、数值、角标、入口数量和排列方向。
- 结构规则：单个功能入口作为基础组件；多个入口使用自动布局组成入口组，不将整组做成不可拆分图片。
- 规范字段：Icon 尺寸、容器尺寸、黑色蒙层透明度、圆角、文字字号、内边距和组间距。
- 推荐圆角：`R3`；后续补充正式 Zero 组件时以组件定稿为准。
- Zero 建议：通过组件属性控制 Icon 替换、文本、角标显隐和状态变体。

## 进度激励区
- 用途：展示收集、助力、累计任务或阶段解锁进度。
- 组件类型：阶段奖励进度 / 目标任务进度。
- 基础组成：标题、进度、奖励节点、状态说明；倒计时和行动按钮按 PRD 选用。
- 节点状态：已完成 / 进行中 / 未完成 / 最终奖励。
- 按钮状态：主操作 / 辅操作 / 禁用。
- 可改项：标题、当前值、目标值、节点数量、奖励内容、截止时间、按钮文案。
- 结构规则：节点数量变化时保持等距分布；节点文案不得与相邻节点重叠。
- 操作规则：一个模块最多突出一个主操作；双按钮时使用主次层级。
- 推荐圆角：外层容器使用 `R5`，内部按钮使用 `R4`。
- Zero 建议：分别建立“阶段奖励进度”和“目标任务进度”组件集，通过节点数量、状态、倒计时和按钮显隐控制变体。

## Feeds
- zeroLink: https://relay.jd.com/file/design?id=2066729196234747906&page_id=0%3A2&node_id=7%3A734
- 用途: Feeds 占位
- 尺寸: 1078x846
- 不可改项: 信息结构
- 推荐圆角：外层容器使用 `R4`；若标准 Zero 组件已有固定圆角，则保持原值。
- 布局：当首屏空间不足时，应整体下移；有资源位时放在资源位之后，无资源位时直接承接前一模块。允许位于首屏外或只露出顶部；不得压缩核心玩法区，不得与前一模块重叠，不得被父容器裁切
