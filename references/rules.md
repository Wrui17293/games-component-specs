# 组件复用规则

## 目标

提供营销互动首页可复用 Zero 组件的复用边界、缩放规则、裁切规则和布局规则。

## 总规则

1. 除特殊标注外，组件均允许等比缩放。
2. 默认使用 `fitMode=contain`。
3. 不允许只修改外框导致内部裁切。
4. 不允许出现半截、裁切或意外溢出。
5. 已有标准 Zero 组件保持原有圆角。
6. 新建或可调整组件必须标明所用圆角等级，圆角等级引用 `games-lofi`。

## 顶部通栏

- 每个首页必选。
- 固定使用叠加关系。
- 不参与页面纵向排布。
- 内部返回、页面名称、奖品入口和更多入口按组件既有结构使用。
- 不在首页布局阶段重新排列。

## 常驻功能信息区

- 承载账户、等级、体力、次数、资产、聊天、商城、兑换等常驻信息或功能入口。
- 固定使用叠加关系。
- 组件自身不参与页面纵向排布。
- 多个入口使用自动布局组成入口组，不将整组做成不可拆分图片。

## 奖励素材

- 京豆、红包、红包带金额等已有组件优先复用。
- 金额、文案或状态可按组件能力调整。
- 福利素材尺寸建议占所在格子的 55%-75%。

## 资源位与 Feeds

- 资源位 Icon 根据 PRD 选择对应状态。
- Feeds 不可修改信息结构。
- 当首屏空间不足时，Feeds 应整体下移。
- Feeds 不得压缩核心玩法区，不得与前一模块重叠，不得被父容器裁切。

## 验收规则

1. 组件比例未变形。
2. 组件内容完整可见。
3. 组件未被父容器裁切。
4. 标准组件未被重复绘制。
5. 圆角、状态、尺寸和可改项符合组件说明。
